# 01. 시스템 아키텍처

## 전체 구성도

```
[외부 소스]
  - GitHub API (CVE POC 검색)
  - CIRCL API (최신 CVE 목록)
  - CVE API (cve.circl.lu, CVE 상세)
       │
       ▼
[Python 수집/분석]
  - main.py (GitHub 수집 오케스트레이션)
  - github_collector.py
  - circl_cve_collector.py
  - cve_info_collector.py / cve_api_collector.py
  - ai_analyzer.py (Gemini)
  - gemini_account_manager.py
  - create_integrated_data.py
  - db_manager.py, config_loader.py, translator.py, file_manager.py
       │
       ▼
[MySQL - TOTORO]
  - Github_CVE_Info, CVE_Info, CVE_Packet_AI_Analysis,
  - CVE_Integrated_Data, users, board_posts, chat_messages, api_tokens 등
       │
       ▼
[웹 서비스]
  - Node Express (server.js) : 32577(HTTP), 32578(HTTPS)
  - React+Vite (client)      : 3000
  - Socket.IO (실시간 채팅)
```

## 데이터 흐름 요약

1. **수집**: GitHub → `Github_CVE_Info` (POC 메타/README). CIRCL → `CVE_Info`. CVE API/Info → `CVE_Info` 보강.
2. **AI 분석**: `Github_CVE_Info` (AI_chk='N') → Gemini 분석 → `CVE_Packet_AI_Analysis`.
3. **통합**: `Github_CVE_Info` + `CVE_Info` + `CVE_Packet_AI_Analysis` → `CVE_Integrated_Data`.
4. **웹**: Express가 MySQL에서 읽어 API 제공 → React에서 대시보드/CVE/POC/게시판/채팅/관리 기능 제공.
