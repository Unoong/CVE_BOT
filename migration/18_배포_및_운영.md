# 18. 배포 및 운영

## 서버 실행 (Windows)

- **PM2 사용**:
  - **서버실행.bat**: PATH에 Node·npm(%APPDATA%\npm) 추가, 포트 3000·32577·32578 정리 후 `pm2 start ecosystem.config.js`.
  - **ecosystem.config.js**: cve-backend(node server.js), cve-frontend(node vite, port 3000). 로그는 프로젝트 상위 `logs/` (backend-out/error, frontend-out/error).
- **직접 실행**: run.bat — 백엔드(node server.js), 프론트(cd client && npm run dev) 각각 CMD 창에서 실행.

## 포트

| 서비스 | 포트 | 비고 |
|--------|------|------|
| 프론트엔드(Vite) | 3000 | strictPort: true |
| 백엔드 HTTP | 32577 | 0.0.0.0 |
| 백엔드 HTTPS | 32578 | 인증서 있을 때 |

## 스케줄

- **로그인 시 서버 자동 실행**: register_schedule_admin.bat 를 **관리자 권한**으로 실행. 작업 이름 `CVE_Bot_Server_AutoStart`, ONLOGON, 실행 파일 `서버실행.bat`.
- **대시보드 통계 갱신**: register_task_scheduler.ps1 를 **관리자 권한** PowerShell에서 실행. 작업 이름 `CVE_Dashboard_Stats_Update`, 매일 03:00, 실행 파일 `update_dashboard_stats.bat`.

## SSL

- **파일**: web/server.cert, web/server.key. 없으면 HTTP만 사용.
- **자체 서명**: generate_self_signed.bat, setup_ssl.bat 등.
- **Let's Encrypt**: renew_letsencrypt_cert.ps1, QUICK_SSL_SETUP.bat, HTTPS_SETUP_GUIDE.md 참고.

## 로그

- **PM2**: 프로젝트 루트 `logs/backend-out.log`, `backend-error.log`, `frontend-out.log`, `frontend-error.log`.
- **Python**: 프로젝트 루트 `logs/` (circl_collector.log, ai_analyzer.log 등).
- **웹 로그 레벨**: web/logger.config.json (info/error/debug).

## 환경 변수 (참고)

- **Node**: SITE_NAME (사이트명). .env 에 두면 server.js 의 dotenv 로 로드.
- **Python**: CVE_BOT_ENV (TEST/PROD/DEV) — config.json / config.test.json / config.prod.json 선택.
- **프론트**: VITE_SITE_NAME (빌드 시 사이트명).

## 마이그레이션 시 체크

1. DB 연결 정보(host, port, user, password, database) 변경.
2. JWT_SECRET, 이메일 계정/앱 비밀번호 변경.
3. Gemini 계정 경로·이메일 목록 변경(gemini_account_manager.py).
4. GitHub 토큰·설정(config.json, 환경변수) 확인.
5. API_URL·SITE_NAME(클라이언트/서버) 확인.
6. SSL 인증서 교체(운영 도메인용).
7. 방화벽에서 3000, 32577, 32578 허용.
8. 작업 스케줄러 작업 경로·실행 파일 경로를 새 서버 경로로 수정.
