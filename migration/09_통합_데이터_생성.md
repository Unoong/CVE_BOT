# 09. 통합 데이터 생성

## 역할

- `Github_CVE_Info`, `CVE_Info`, `CVE_Packet_AI_Analysis`를 조인해 하나의 테이블 `CVE_Integrated_Data`로 생성/갱신. 웹/API에서 CVE+POC+AI 결과를 한 번에 조회하기 위함.

## 파일

- **create_integrated_data.py**: 통합 테이블 생성 및 데이터 채우기/갱신.

## CVE_Integrated_Data 구조 요약

- **hash_key**: `sha256(github_cve + github_link + github_date + ai_step)` 등으로 생성해 행 유일성 보장.
- **github_***: GitHub POC 쪽 필드(github_cve, github_link, github_title, github_writer, github_date, github_readme, github_trans_msg, github_download_path, github_status 등).
- **cve_***: CVE_Info 쪽 필드(cve_code, cve_state, cve_date_published, cve_product, cve_descriptions, cve_cvss_score, cve_cvss_severity, cve_solutions 등).
- **ai_***: AI 분석 쪽 필드(ai_step, ai_packet_text, ai_vuln_stage, ai_stage_description, ai_expected_response, ai_mitre_tactic, ai_mitre_technique, ai_snort_rule, ai_cve_summary, ai_remediation 등).

## 처리 방식

- 기존 통합 테이블이 있으면 truncate 또는 조건에 따라 삭제 후, 3개 소스 조인 결과를 INSERT.
- CVE당·POC당·단계당 행이 하나씩 될 수 있도록 hash_key 설계.

## 실행

- `python create_integrated_data.py` (수집/AI 분석 후 주기 실행하거나, 대시보드 갱신 전에 실행).
