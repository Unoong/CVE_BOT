# 04. 수집 모듈 개요

## 파이프라인 요약

1. **GitHub POC 수집** (`main.py` + `github_collector.py`)
   - GitHub API로 CVE 키워드 검색 → 저장소 메타·README 수집 → `Github_CVE_Info` 저장, POC zip 다운로드(`file_manager`).

2. **CIRCL CVE 수집** (`circl_cve_collector.py`)
   - CIRCL Vulnerability-Lookup API(`/last`)로 최신 CVE(CSAF) 목록 수집 → `CVE_Info`에 적재.
   - 실행: `run_circl_collector.bat` 또는 직접 `circl_cve_collector.py`.

3. **CVE 상세 정보 수집** (`cve_info_collector.py` / CVE API)
   - `Github_CVE_Info`의 CVE 코드 등에 대해 `cve.circl.lu/api/cve/{cve_code}` 호출 → 파싱 후 `CVE_Info` 보강/삽입.
   - `cve_api_collector.py`: 통합 수집·전송 스크립트 내 CVE API 사용.

4. **AI 분석** (`ai_analyzer.py` + `run_ai_analysis.py`)
   - `Github_CVE_Info`에서 `AI_chk='N'`인 항목 대상, Gemini CLI로 POC 분석 → `CVE_Packet_AI_Analysis`에 단계별 저장, `AI_chk` 업데이트.
   - 계정 로테이션: `gemini_account_manager.py`.

5. **통합 데이터 생성** (`create_integrated_data.py`)
   - `Github_CVE_Info` + `CVE_Info` + `CVE_Packet_AI_Analysis` 조인 → `CVE_Integrated_Data` 생성/갱신(hash_key 기준).

## 설정 의존

- **Python**: 루트 `config.json`(또는 `config.test.json`/`config.prod.json`) — DB, 수집 옵션.
- **GitHub**: 설정/환경에서 GitHub 토큰(또는 토큰 목록) 필요.
- **AI 분석**: `ai_analysis_config.json`, `prompt.txt`/`prompt_instruction.txt`, Gemini 계정 경로/목록(`gemini_account_manager.py`).
