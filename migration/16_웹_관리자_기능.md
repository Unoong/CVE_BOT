# 16. 웹 관리자 기능

## 접근 조건

- JWT 인증 + 역할 `admin` (일부는 `analyst`도 가능).

## 사용자 관리

- **API**:
  - **GET /api/admin/users**: 사용자 목록.
  - **PUT /api/admin/users/:id/role**: 역할 변경(user/analyst/admin).
  - **PUT /api/admin/users/:id/reset-password**: 비밀번호 초기화(임시 비밀번호 설정).
  - **PUT /api/admin/users/:id/nickname**: 닉네임 변경(관리자 대신 수정).
  - **DELETE /api/admin/users/:id**: 사용자 삭제.

- **페이지**: AdminPanel.jsx 내 사용자 관리 섹션.

## 로그

- **GET /api/admin/logs**: 로그 파일 목록 및 내용 조회(로그 경로, 파일명은 서버 설정에 따름). 페이지네이션/필터 가능.
- **페이지**: LogsView.jsx

## API 토큰

- **POST /api/admin/tokens**: 토큰 생성(name, permissions, expires_at).
- **GET /api/admin/tokens**: 목록 조회.
- **DELETE /api/admin/tokens/:id**: 삭제.
- **PATCH /api/admin/tokens/:id/status**: 활성/비활성 전환.
- **페이지**: ApiTokens.jsx

## 시스템 설정

- **GET/PUT /api/admin/system-config/collection**: 수집 관련 설정(웹에서 보여주는 용도 또는 수집 스크립트용 설정 저장).
- **GET/PUT /api/admin/system-config/ai-analysis**: AI 분석 설정(병렬, 제한 등).
- **GET/PUT /api/admin/system-config/default-role**: 가입 시 기본 역할.
- **GET/PUT /api/admin/cve-limits**: CVE별 POC 수 제한(전역 기본값, CVE별 오버라이드).
- **페이지**: SystemConfig.jsx, CVEConfig.jsx 등.

## DB 쿼리

- **POST /api/db/query**: SQL 실행(읽기 전용 또는 제한된 쓰기). analyst, admin만.
- **페이지**: DBQuery.jsx
- **보안**: SELECT 위주로 제한, 위험 구문 필터링 권장.
