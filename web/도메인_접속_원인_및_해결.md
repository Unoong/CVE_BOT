# https://www.ai-platform.store:3000 접속 안 될 때

## 현상

- **IP로만 접속됨**: `https://192.168.45.244:3000` 또는 공인 IP로는 접속 가능
- **도메인으로는 안 됨**: `https://www.ai-platform.store:3000` 접속 실패

---

## 원인 (포트포워딩이 아니라 NAT 루프백)

**같은 공유기(내부 네트워크) 안**에서 접속할 때:

1. `www.ai-platform.store` → DNS 조회 → **공인 IP**로 응답
2. 브라우저가 **공인 IP:3000**으로 접속 시도
3. 많은 **가정용/소호 공유기는 NAT 루프백(헤어핀 NAT)을 지원하지 않음**
4. 그래서 “내부 PC → 공인 IP” 요청이 공유기에서 **다시 내부 서버로 돌려주지 못함** → 연결 실패

즉, **포트포워딩은 정상**이고, **같은 네트워크 안에서 공인 IP/도메인으로 접속하는 구간**에서 막히는 경우가 많습니다.

---

## 확인 방법

1. **진단 스크립트 실행**  
   `web` 폴더에서 **도메인_접속_진단.bat** 실행  
   - `www.ai-platform.store`가 어떤 IP로 잡히는지  
   - 이 PC의 내부 IP  
   - 공인 IP  
   를 한 번에 확인할 수 있습니다.

2. **외부에서 테스트**  
   - 휴대폰 **Wi‑Fi 끄고 4G/5G만** 사용  
   - 브라우저에서 `https://www.ai-platform.store:3000` 접속  
   - 여기서 **접속되면** → 포트포워딩·도메인 모두 정상, 문제는 **내부에서의 NAT 루프백**입니다.

---

## 해결 방법

### 1) 같은 네트워크 안에서는 내부 IP로 접속 (가장 간단)

- 내부(집/사무실)에서는 **`https://192.168.45.244:3000`** 처럼 **내부 IP + 포트**로 접속
- 외부에서는 **`https://www.ai-platform.store:3000`** 사용

### 2) 공유기에서 NAT 루프백(헤어핀 NAT) 켜기

- 공유기 관리 페이지 → 포트포워딩/가상서버/고급 설정 등에서  
  **NAT 루프백**, **헤어핀 NAT**, **NAT Reflection** 옵션이 있으면 **사용**으로 설정
- 기기/펌웨어에 따라 없을 수 있음

### 3) 내부용 DNS (Split-Horizon DNS)

- **같은 공유기 안**에서만 `www.ai-platform.store` → **192.168.45.244** 로 풀리게 설정
  - 공유기 DNS 설정에서 호스트 매핑(도메인 → 192.168.45.244)  
  - 또는 PC의 `hosts` 파일에  
    `192.168.45.244  www.ai-platform.store`  
    추가
- 그러면 내부에서는 도메인으로 접속해도 **내부 IP로 바로 연결**되어 NAT 루프백 없이 동작

### 4) DNS A 레코드가 현재 공인 IP와 다른 경우 (공유기 변경 시 자주 발생)

- **현재 확인된 값**
  - `www.ai-platform.store` → **114.108.49.125** (DNS에 등록된 IP)
  - 이 PC에서 나가는 공인 IP → **211.208.47.4**
- 공유기를 바꾸거나 회선이 바뀌면 **공인 IP가 바뀌는데**, 도메인 DNS를 새 공인 IP로 갱신하지 않으면 도메인으로 접속이 안 됨.
- **조치**: 도메인 관리처(가비아, 클라우드플레어 등)에서 **www.ai-platform.store A 레코드를 현재 공인 IP(211.208.47.4)** 로 수정.
- 수정 후 전파에는 수 분~몇 시간 걸릴 수 있음.

## 정리

| 접속 위치     | 사용 주소                           | 비고                    |
|--------------|-------------------------------------|-------------------------|
| 내부(같은 공유기) | `https://192.168.45.244:3000`       | 도메인 대신 IP 사용     |
| 외부         | `https://www.ai-platform.store:3000` | 포트포워딩·DNS 정상이면 가능 |

**IP로만 접속되고 도메인으로는 안 되는 경우**, 포트포워딩 문제보다 **NAT 루프백(헤어핀)** 또는 **내부 DNS 설정** 문제인 경우가 많습니다.
