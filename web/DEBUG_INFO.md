# 디버깅 로그 가이드

## 🔍 로깅이 추가된 위치

### 백엔드 (server.js)
모든 회원가입 단계에 상세 로그 추가:

1. **이메일 인증 코드 발송** (`/api/auth/send-verification`)
   - [1] 요청 이메일
   - [2] 이메일 중복 확인
   - [3] 생성된 인증 코드
   - [4] 인증 코드 저장 완료 + 만료 시각
   - [5] 이메일 발송 시작/완료
   - [6] 현재 저장된 인증 코드 목록

2. **인증 코드 확인** (`/api/auth/verify-code`)
   - [1] 확인 요청 (이메일, 코드)
   - [2] 저장된 데이터 확인
   - [3] 현재 시각, 만료 시각, 남은 시간
   - [4] 코드 비교 (입력 vs 저장)
   - [5] 인증 성공/실패

3. **회원가입** (`/api/auth/register`)
   - [1] 회원가입 정보 (아이디, 이름, 이메일, 전화번호, 인증코드)
   - [2] 인증 코드 확인
   - [3] 사용자ID 중복 확인
   - [4] 비밀번호 암호화
   - [5] 데이터베이스에 사용자 생성
   - [6] 인증 코드 삭제
   - [7] 남은 인증 코드 목록
   - [8] 회원가입 완료

### 프론트엔드 (Register.jsx)
브라우저 콘솔에 상세 로그 추가:

1. **인증 코드 발송**
   - [1] 입력된 이메일
   - [2] 서버 요청
   - [3] 서버 응답
   - [4] 타이머 시작

2. **인증 코드 확인**
   - [1] 입력값 (이메일, 코드, 남은 시간)
   - [2] 서버 확인 요청
   - [3] 서버 응답
   - [4] 이메일 인증 완료
   - [5-6] 다음 단계 이동

3. **회원가입**
   - [1] 입력된 정보 (아이디, 이름, 이메일, 전화번호, 비밀번호 길이, 인증 완료 여부)
   - [2] 유효성 검사 통과
   - [3] 서버 요청
   - [4] 서버 응답
   - [5] 가입 완료 단계 이동
   - [6-7] 로그인 페이지 이동

4. **타이머**
   - 30초마다 남은 시간 출력
   - 만료 시 상태 초기화

## 📋 디버깅 방법

### 1. 백엔드 로그 확인
서버를 실행한 CMD 창에서 로그를 확인하세요:

```
========== 이메일 인증 코드 발송 시작 ==========
[1] 요청 이메일: test@example.com
[2] 이메일 중복 확인 중...
[2] 기존 사용자 수: 0
[3] 생성된 인증 코드: 123456
[4] 인증 코드 저장 완료 (유효시간: 3분)
[4] 만료 시각: 2025. 10. 11. 오후 10:45:30
[5] 이메일 발송 시작...
[5] 발신: totoro.cve.bot@gmail.com
[5] 수신: test@example.com
[6] ✅ 이메일 발송 완료
[7] 현재 저장된 인증 코드 목록: [ 'test@example.com' ]
========================================
```

### 2. 프론트엔드 로그 확인
브라우저 개발자 도구 (F12) → Console 탭에서 확인:

```
========== [프론트] 인증 코드 발송 시작 ==========
[1] 입력된 이메일: test@example.com
[2] 서버에 요청 보내는 중... http://localhost:32577/api/auth/send-verification
[3] ✅ 서버 응답: {message: "인증 코드가 발송되었습니다"}
[4] 타이머 시작: 180초 (3분)
========================================
```

### 3. 에러 발생 시
에러가 발생하면 다음 정보가 출력됩니다:

**백엔드:**
```
❌ [에러 발생]
에러 메시지: ...
에러 스택: ...
========================================
```

**프론트엔드:**
```
❌ [프론트] 에러 발생:
에러 응답: {...}
에러 메시지: ...
========================================
```

## 🐛 문제 해결 체크리스트

### 이메일 발송이 안될 때
1. 백엔드 로그에서 `[5] 이메일 발송 시작...` 이후 에러 확인
2. Gmail 앱 비밀번호 확인: `nqvc vhwq lium gmmn`
3. 인터넷 연결 확인

### 인증 코드가 일치하지 않을 때
1. 백엔드 로그에서 `[4] 코드 비교` 부분 확인
2. 입력한 코드와 저장된 코드 비교
3. 공백이나 대소문자 확인

### 인증 코드가 만료될 때
1. 프론트엔드 콘솔에서 `⏰ [타이머]` 로그 확인
2. 백엔드 로그에서 `[3] 남은 시간` 확인
3. 3분(180초) 이내에 인증 완료해야 함

### 회원가입이 실패할 때
1. 백엔드 로그에서 어느 단계에서 실패했는지 확인
2. 프론트엔드 콘솔에서 입력 데이터 확인
3. 에러 메시지 확인

## 💡 팁
- 백엔드 로그와 프론트엔드 로그를 **동시에** 확인하면 문제를 빠르게 찾을 수 있습니다
- 각 단계마다 ✅ 표시가 나오는지 확인하세요
- 에러 발생 시 스택 트레이스를 확인하여 정확한 위치 파악
