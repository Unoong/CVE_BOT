# CVE BOT 웹서버 운영 가이드

## 🚨 문제: CMD 창이 멈추는 현상

### 원인
Windows CMD 창의 **QuickEdit 모드** 때문에 발생합니다.
- 마우스로 CMD 창을 클릭하면 "선택 모드"로 전환
- 프로그램이 일시 중지됨
- 엔터를 누르면 다시 실행

---

## ✅ 해결 방법

### 1️⃣ **QuickEdit 모드 비활성화 (권장)**

```batch
# 관리자 권한으로 실행
disable_quickedit.bat
```

**적용 후:**
- 새로운 CMD 창부터 적용됨
- 마우스 클릭해도 멈추지 않음
- 텍스트 선택은 타이틀바 우클릭 → 편집 → 표시로 가능

---

### 2️⃣ **PM2로 백그라운드 서비스 실행 (최고 권장) ⭐**

#### 설치 및 실행
```batch
# 웹서버를 백그라운드 서비스로 실행
cd E:\LLama\pythonProject\CVE_BOT\web
start_service.bat
```

#### 장점
- ✅ CMD 창 필요 없음
- ✅ 자동 재시작 (크래시 시)
- ✅ 부팅 시 자동 시작
- ✅ 로그 관리
- ✅ 멀티 프로세스 지원
- ✅ 여러 사용자 동시 접속 가능

#### 주요 명령어
```bash
# 상태 확인
pm2 status

# 로그 확인 (실시간)
pm2 logs cve-bot-server

# 로그 확인 (최근 100줄)
pm2 logs cve-bot-server --lines 100

# 서버 재시작
pm2 restart cve-bot-server

# 서버 중지
pm2 stop cve-bot-server

# 서버 시작
pm2 start cve-bot-server

# 서버 삭제
pm2 delete cve-bot-server

# 모든 서버 상태
pm2 list

# 모니터링 (CPU, 메모리)
pm2 monit
```

## 🔧 성능 최적화

### DB 연결 풀 설정
현재 설정 (server.js):
```javascript
connectionLimit: 100,        // 최대 100개 동시 연결
connectTimeout: 10000,       // 연결 타임아웃 10초
acquireTimeout: 10000,       // 획득 타임아웃 10초
idleTimeout: 60000,          // 유휴 연결 60초 후 해제
enableKeepAlive: true        // Keep-Alive 활성화
```

### 권장 사항
1. **MySQL 최대 연결 수 확인**
   ```sql
   SHOW VARIABLES LIKE 'max_connections';
   -- 기본값: 151 (필요 시 증가)
   
   SET GLOBAL max_connections = 200;
   ```

2. **서버 메모리 모니터링**
   ```bash
   pm2 monit
   ```

## 📊 동시 접속자 처리

### Node.js 비동기 특성
- **단일 스레드**지만 **이벤트 루프**로 동작
- I/O 작업 (DB 쿼리) 시 블로킹 없음
- 수천 명 동시 접속 가능

### 예시
```
사용자 A: CVE 검색 (2초 소요)
사용자 B: 대시보드 조회 (1초 소요)
사용자 C: CVE 상세 조회 (0.5초 소요)

→ 3개 요청이 동시에 처리됨 ✅
→ 서로 블로킹하지 않음 ✅
```

### 병목 현상 발생 시
1. **DB 쿼리 최적화**
   - 인덱스 추가
   - 쿼리 실행 계획 분석 (`EXPLAIN`)
   
2. **연결 풀 증가**
   ```javascript
   connectionLimit: 200  // 100 → 200
   ```

3. **캐싱 도입**
   - Redis 사용
   - 통계 데이터 캐싱

## 🆘 문제 해결

### 1. 서버가 시작 안 됨
```bash
# 로그 확인
pm2 logs cve-bot-server --err

# 포트 사용 중 확인
netstat -ano | findstr :32577
```

### 2. DB 연결 실패
```bash
# MySQL 서비스 확인
net start | findstr MySQL

# MySQL 재시작
net stop MySQL80
net start MySQL80
```

### 3. 메모리 부족
```bash
# 프로세스 재시작
pm2 restart cve-bot-server

# 메모리 모니터링
pm2 monit
```

### 4. 로그 파일 너무 큼
```bash
# 로그 삭제
pm2 flush

# 특정 서버 로그만 삭제
pm2 flush cve-bot-server
```
