# 로그 레벨 설정 가이드

## 📋 개요
이 시스템은 3단계 로그 레벨을 지원하여 필요에 따라 출력되는 정보량을 조절할 수 있습니다.

---

## 🎚️ 로그 레벨 종류

### 1. **info** (기본)
- **출력 내용**: 서버 시작/종료, API 요청 등 기본 정보만
- **사용 시기**: 운영(Production) 환경
- **예시 출력**:
  ```
  [INFO] ========== 로그인 시도 ==========
  [INFO] [6] 로그인 성공 - 사용자: admin
  ```

### 2. **error** (에러 포함)
- **출력 내용**: 에러 메시지 + 기본 정보
- **사용 시기**: 에러 추적이 필요할 때
- **예시 출력**:
  ```
  [INFO] ========== 로그인 시도 ==========
  [ERROR] [3] 사용자를 찾을 수 없음
  [ERROR] [로그인 에러] Invalid credentials
  ```

### 3. **debug** (상세 디버그)
- **출력 내용**: 모든 단계별 실행 내용 상세 출력
- **사용 시기**: 개발 중, 버그 수정, 문제 진단
- **예시 출력**:
  ```
  [INFO] ========== 로그인 시도 ==========
  [DEBUG] [1] 로그인 요청 - 아이디: admin
  [DEBUG] [2] DB에서 사용자 조회 중...
  [DEBUG] [3] 사용자 발견, 비밀번호 확인 중...
  [DEBUG] [4] 비밀번호 확인 완료
  [DEBUG] [5] JWT 토큰 생성 중...
  [INFO] [6] 로그인 성공 - 사용자: admin
  ```

## ⚙️ 백엔드 로그 레벨 변경 방법

### 1단계: 설정 파일 열기
```
E:\LLama\pythonProject\CVE_BOT\web\logger.config.json
```

### 2단계: logLevel 값 변경
```json
{
  "logLevel": "debug",    ← 여기를 변경! (info / error / debug)
  "description": { ... }
}
```

### 3단계: 서버 재시작
- 백엔드 서버 창에서 `Ctrl+C`
- 다시 실행: `node server.js`

## 🎨 프론트엔드 로그 레벨 변경 방법

```

```

### 2단계: LOG_LEVEL 상수 변경

- 프론트엔드 서버 창에서 `Ctrl+C`

## 💡 권장 사용법

| 상황 | 권장 레벨 | 이유 |
|------|----------|------|
| 개발 중 | **debug** | 모든 단계를 확인하여 문제 빠르게 파악 |
| 버그 수정 | **debug** | 정확한 오류 지점 특정 |
| 테스트 | **error** | 기본 흐름 + 에러만 확인 |
| 운영(Production) | **info** | 최소한의 정보만 출력하여 성능 유지 |

## ❗ 주의사항

1. **보안**: 운영 환경에서는 `info` 레벨 사용 권장 (비밀번호 등 민감 정보 노출 방지)
2. **성능**: `debug` 레벨은 많은 로그를 출력하므로 운영 환경에서는 피할 것
3. **파일 위치**: 설정 파일을 잃어버리지 않도록 주의

## 📞 문제 해결

### 로그가 출력되지 않을 때
1. 설정 파일이 올바른 위치에 있는지 확인
2. JSON 문법이 올바른지 확인
3. 서버를 재시작했는지 확인

### 로그가 너무 많을 때
- 레벨을 `debug` → `error` 또는 `info`로 변경

### 특정 로그만 보고 싶을 때
- 브라우저 개발자 도구에서 필터링 사용 (F12 → Console → Filter)
