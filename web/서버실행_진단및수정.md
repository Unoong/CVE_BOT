# 서버 실행·스케줄 등록 문제 진단 및 수정 요약

## 발견된 원인

### 1. 서버실행.bat 실행 시 "pm2를 찾을 수 없음"
- **원인**: 배치를 더블클릭으로 실행하면 **PATH에 `%APPDATA%\npm`이 없음**. pm2는 보통 여기에 설치됨.
- **수정**: `서버실행.bat`에서 `set "PATH=%APPDATA%\npm;%PATH%"` 추가.

### 2. 접속 안 됨 / EADDRINUSE (포트 사용 중)
- **원인**: 배치가 **포트 3000만** 정리하고, 백엔드 포트 **32577, 32578**은 정리하지 않음. 이전 프로세스가 남아 있으면 새 백엔드가 기동 실패.
- **수정**: 포트 **3000, 32577, 32578** 모두 사용 중인 프로세스를 종료하도록 변경. 재실행 전 2초 대기 추가.

### 3. 프론트엔드(NPM.CMD SyntaxError)
- **원인**: PM2가 Windows에서 프론트엔드를 기동할 때 `node` 대신 `npm.cmd`를 스크립트로 실행하는 경우가 있음.
- **수정**: `ecosystem.config.js`에서 프론트엔드 앱에 `interpreter: "node"`, `script: "node_modules/vite/bin/vite.js"` 로 명시 (args 제거).

### 4. 스케줄 등록 후에도 서버가 안 켜짐
- **원인**: **ONSTART**(부팅 시)에는 사용자 프로필/PATH가 없어서 pm2·node를 찾지 못함.
- **수정**: `register_schedule_admin.bat`에서 **ONLOGON**(사용자 로그온 시)으로 변경. 로그인 시 자동 실행되며 PATH가 정상 적용됨.

---

## 수정된 파일

| 파일 | 변경 내용 |
|------|-----------|
| `web\서버실행.bat` | PATH에 `%APPDATA%\npm` 추가, 포트 32577·32578 정리 추가, 대기 2초 |
| `web\ecosystem.config.js` | 프론트엔드 `interpreter` + `script` 방식으로 변경 |
| `web\register_schedule_admin.bat` | ONSTART → ONLOGON, 메시지 정리 |

---

## 사용 방법

### 서버 실행
1. **`web\서버실행.bat`** 더블클릭.
2. 처음이라면 **`pm2_설치_및_로그설정.bat`** 를 관리자 권한으로 한 번 실행해 두기.
3. 브라우저에서 **http://localhost:3000** 접속.

### 스케줄 등록 (로그인 시 자동 실행)
1. **`web\register_schedule_admin.bat`** 를 **우클릭 → 관리자 권한으로 실행**.
2. "스케줄 등록 완료" 메시지 확인.
3. 이후 Windows 로그인할 때마다 서버가 자동으로 실행됨.

### 문제 발생 시
- **pm2 상태**: `pm2 status`
- **로그**: `E:\LLama\pythonProject\CVE_BOT\logs\` 아래 `backend-out.log`, `backend-error.log`, `frontend-out.log`, `frontend-error.log`
- **포트 점유 확인**: `netstat -ano | findstr "3000 32577 32578"`
