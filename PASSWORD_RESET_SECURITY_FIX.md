# 🔒 비밀번호 재설정 보안 취약점 수정

## ⚠️ 문제점

### 기존 시스템의 심각한 보안 취약점
```
사용자가 아이디와 이메일만 알면 → 즉시 비밀번호 변경 가능!
→ 누구나 다른 사람의 비밀번호를 마음대로 바꿀 수 있음!
```

**예시 공격 시나리오**:
```
1. 공격자가 피해자의 아이디를 알아냄
2. 피해자의 이메일 주소를 추측 또는 소셜 엔지니어링으로 획득
3. "비밀번호 재설정" 기능 악용
4. ❌ 피해자 계정 탈취 성공!
```

---

## ✅ 해결 방법

### 3단계 보안 인증 프로세스 도입

```
1단계: 본인 확인 (아이디 + 이메일)
   ↓
2단계: 이메일 인증 코드 발송 및 확인 (3분 유효)
   ↓
3단계: 인증 성공 후 새 비밀번호 설정
```

**이제는 이메일 소유권을 증명해야만 비밀번호 변경 가능!** 🔐

## 🔧 수정 사항

### 1. Backend (server.js)

#### 인증 코드 저장소 추가
```javascript
// Line 40
const passwordResetCodes = {}; // 비밀번호 재설정 인증 코드
```

#### 3개의 새로운 API 엔드포인트

**1) 인증 코드 발송**
```javascript
POST /api/auth/reset-password-send-code
```
- 아이디와 이메일로 사용자 확인
- 6자리 랜덤 코드 생성
- 이메일 발송
- 3분 유효시간

**2) 인증 코드 확인**
```javascript
POST /api/auth/reset-password-verify-code
```
- 이메일 + 코드 검증
- 만료 시간 확인
- 코드 일치 여부 확인

**3) 비밀번호 변경 (인증 완료 후)**
```javascript
POST /api/auth/reset-password
```
- 인증 코드 **재확인** (보안 강화)
- 사용자 정보 확인
- 비밀번호 해싱 후 저장
- 인증 코드 삭제

## 🔐 보안 강화 포인트

### 1. 이중 인증 확인
```javascript
// 비밀번호 변경 시 인증 코드를 한 번 더 확인
if (stored.code !== code || stored.username !== username) {
    return res.status(400).json({ error: '인증 정보가 일치하지 않습니다' });
}
```

### 2. 시간 제한
```javascript
// 3분 후 자동 만료
const expiry = Date.now() + 3 * 60 * 1000;

if (Date.now() > stored.expiry) {
    delete passwordResetCodes[email];
    return res.status(400).json({ error: '인증 코드가 만료되었습니다' });
}
```

### 3. 인증 코드 일회성
```javascript
// 비밀번호 변경 성공 후 즉시 삭제

```

### 4. 상세한 로깅
