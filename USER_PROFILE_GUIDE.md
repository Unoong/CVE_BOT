# 👤 사용자 프로필 기능 완료

## ✅ 완료된 작업

### 1. 백엔드 API (✅)
- **`GET /api/users/profile`**: 사용자 본인 정보 조회
- **`PUT /api/users/profile/nickname`**: 닉네임 변경
- **`PUT /api/users/profile/password`**: 비밀번호 변경

### 2. 프론트엔드 (✅)
- **Profile.jsx**: 프로필 페이지 생성
- **App.jsx**: 라우트 추가 (`/profile`)
- **Layout.jsx**: 사용자 메뉴에 "내 프로필" 링크 추가

## 🚀 사용 방법

### 1. 프로필 페이지 접속

**방법 1: 사용자 메뉴**
```
우측 상단 아바타 클릭 → "내 프로필" 클릭
```

**방법 2: 직접 URL**
```
http://114.108.49.125:3000/profile
```

### 2. 닉네임 변경

1. **프로필 페이지** 접속
2. **기본 정보** 카드에서 **닉네임** 항목 찾기
3. **변경** 버튼 클릭
4. **새 닉네임 입력** (최소 2자)
5. **변경** 버튼 클릭
6. ✅ 성공 메시지 확인

### 3. 비밀번호 변경

1. **프로필 페이지** 접속
2. **보안 설정** 카드에서 **비밀번호 변경** 버튼 클릭
3. **현재 비밀번호** 입력
4. **새 비밀번호** 입력 (최소 4자)
5. **새 비밀번호 확인** 입력
6. **변경** 버튼 클릭
7. ✅ 성공 메시지 확인
8. 🔄 3초 후 자동 로그아웃 → 다시 로그인 필요

## 🔧 API 상세

### 1. 프로필 조회

**엔드포인트**
```
GET /api/users/profile
```

**요청 헤더**
```
Authorization: Bearer {JWT_TOKEN}
```

**응답**
```json
{
  "id": 1,
  "username": "user1",
  "name": "홍길동",
  "nickname": "홍길동",
  "email": "test@example.com",
  "phone": "010-1234-5678",
  "role": "user",
  "created_at": "2025-10-11T12:00:00.000Z"
}
```

## 📝 주의사항

### 1. 닉네임 변경
- **즉시 반영**: 변경 즉시 채팅/게시판에 적용
- **중복 허용**: 다른 사용자와 같은 닉네임 가능
- **빈 값 불가**: 최소 2자 이상 필수
- **특수문자**: 사용 가능

### 2. 비밀번호 변경
- **현재 비밀번호 필수**: 보안을 위해 확인
- **자동 로그아웃**: 변경 후 3초 뒤 로그아웃
- **다시 로그인 필요**: 새 비밀번호로 로그인
- **최소 길이**: 4자 이상 (8자 이상 권장)

### 3. 권한
- **본인만 변경 가능**: 다른 사용자 프로필 수정 불가
- **관리자 권한**: 관리자는 관리자 페이지에서 모든 사용자 관리 가능

## 🔍 디버깅 팁

### 프로필이 로드되지 않을 때
1. **JWT 토큰 확인**: 로컬 스토리지의 `token` 확인
2. **네트워크 탭**: F12 → Network → `/api/users/profile` 요청 확인
3. **백엔드 로그**: server.js 콘솔에서 에러 확인

### 닉네임 변경이 안 될 때
1. **최소 길이**: 2자 이상인지 확인
2. **API 응답**: 브라우저 콘솔에서 에러 메시지 확인
3. **DB 확인**: users 테이블의 nickname 컬럼 확인

### 비밀번호 변경이 안 될 때
1. **현재 비밀번호**: 정확한지 확인
2. **새 비밀번호**: 최소 4자 이상인지 확인
3. **확인 입력**: 두 번 입력한 비밀번호가 일치하는지 확인
